# VectorVault

> [Español](./README.es.md)

Upload images and search them by meaning. VectorVault uses embeddings to find images visually similar to a text query.

## Status

Under development, only the database schema and migrations exist so far.

---

## Database

Postgres with the [pgvector](https://github.com/pgvector/pgvector) extension. The schema has 3 tables:

### `assets`
Represents an image uploaded to the system.

| Column | Type | Description |
|---|---|---|
| `id` | UUID | PK |
| `storage_path` | String | Path on disk where the image is stored |
| `original_filename` | String | Original file name |
| `bytes` | BigInteger | File size in bytes |
| `created_at` | Timestamp | Upload date |

### `embedding_models`
Tracks the models used to generate embeddings. Allows associating each vector with the model that produced it.

| Column | Type | Description |
|---|---|---|
| `id` | UUID | PK |
| `provider` | String | e.g. `"openclip"` |
| `name` | String | e.g. `"ViT-B-32"` |
| `pretrained` | String | Checkpoint name, e.g. `"laion2b_s34b_b79k"` |
| `dim` | Integer | Vector dimension |
| `created_at` | Timestamp | Registration date |

> The combination `(provider, name, pretrained)` is unique.

### `embeddings`
Links an asset to its vector, generated by a specific model.

| Column | Type | Description |
|---|---|---|
| `id` | UUID | PK |
| `asset_id` | UUID | FK → `assets.id` (CASCADE) |
| `model_id` | UUID | FK → `embedding_models.id` (RESTRICT) |
| `vector` | Vector(512) | Generated embedding |
| `created_at` | Timestamp | Generation date |

> The combination `(asset_id, model_id)` is unique — one embedding per model per asset.  
> An HNSW index on `vector` with cosine distance enables efficient similarity search.

## Running the database

```bash
docker compose up -d
```

Run migrations:

```bash
cd backend
alembic upgrade head
```
